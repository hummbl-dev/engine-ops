# Engine Ops Project Structure

This document provides a comprehensive overview of the Engine Ops directory structure.

## Root Directory

```
engine-ops/
├── core/              # Core business logic
├── public/            # API server and routes
├── infra/             # Infrastructure-as-Code
├── docs/              # Documentation
├── dashboard/         # Admin monitoring UI
├── tools/             # CLI utilities
├── tests/             # Test suites
├── schemas/           # Data schemas and migrations
├── k8s-operator/      # Kubernetes operator
├── sdks/              # Client SDKs (Go, Python)
├── examples/          # Usage examples
└── coverage/          # Test coverage reports
```

## Key Directories Explained

### Application Code

**`core/`** - Business Logic

- AI integration (`ai/clients/`)
- Algorithms (`algorithms/`)
- Anomaly detection (`anomaly/`)
- Credentials management (`infra/credentials/`)
- Observability (`observability/`)
- Prompt templates (`prompts/`)

**`public/`** - API Layer

- Express server setup
- REST API routes
- WebSocket handlers
- Middleware (auth, logging, metrics)

**`schemas/`** - Data Models

- Zod validation schemas
- Database migration scripts

### Infrastructure

**`infra/`** - Deployment & Infrastructure

- **`terraform/`** - GKE cluster provisioning
- **`helm/`** - Kubernetes charts (app + monitoring)
- **`argocd/`** - GitOps configuration
- **`k8s/`** - Raw Kubernetes manifests
- **`scripts/`** - Automation scripts

**`k8s-operator/`** - Kubernetes Controller

- Custom resource definitions (CRDs)
- Workload optimization controller

### Tooling & SDKs

**`tools/`** - CLI Utilities

- `summarize-logs.ts` - Log summarization
- `verify-*.ts` - Validation scripts
- `license-headers/` - License management

**`sdks/`** - Client Libraries

- `go/` - Go SDK
- `python/` - Python SDK

**`dashboard/`** - Admin UI

- Real-time monitoring dashboard
- Chart.js visualizations
- Standalone HTML/CSS/JS app

### Documentation

**`docs/`** - Comprehensive Docs

- **`api/`** - Auto-generated API docs
- **`context-engineering/`** - SRE best practices
- **`prompt-engineering/`** - AI/LLM guidelines
- **`playbooks/`** - Operational runbooks
- **`planning/`** - Historical planning docs
- **`reports/`** - Project reports

### Testing & Quality

**`tests/`** - Test Resources

- `fixtures/` - Test data files
- Integration test helpers

**`coverage/`** - Test Coverage

- Generated by Jest
- HTML reports + LCOV data

### Examples

**`examples/`** - Usage Examples

- Kubernetes deployment examples
- ML plugin integration example

## Important Files

### Configuration

- `.env.example` - Environment variables template
- `package.json` - Dependencies and scripts
- `tsconfig.json` - TypeScript configuration
- `.eslintrc.json` - Linting rules
- `jest.config.js` - Test configuration

### Documentation

- `README.md` - Project overview

- `LICENSE` - BSL 1.1 license

## Navigation Guide

### "I want to..."

**...add a new AI provider**
→ `core/ai/clients/` + update `core/ai/client.ts`

**...modify API endpoints**
→ `public/routes/`

**...change deployment config**
→ `infra/helm/engine-ops/`

**...update observability metrics**
→ `core/observability/metrics.ts`

**...add a new prompt**
→ `core/prompts/<feature-name>/v1.md`

**...modify the dashboard**
→ `dashboard/public/`

**...write operational docs**
→ `docs/playbooks/`

**...manage secrets/credentials**
→ `core/infra/credentials/`

## Design Principles

1. **Separation of Concerns** - Core logic isolated from API/infrastructure
2. **Interface-Driven** - All major components expose interfaces
3. **Multi-Provider** - Support multiple backends (AI, secrets, cloud)
4. **Infrastructure as Code** - All infrastructure declarative
5. **Documentation First** - Comprehensive docs at every level
